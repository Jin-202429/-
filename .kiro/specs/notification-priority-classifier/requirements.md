# Requirements: AI Notification Priority Classifier

## 1. 목적 (What)
- 다양한 플랫폼(이메일, 슬랙, SMS 등)에서 수신되는 알림 메시지를 AI가 분석하여 우선순위를 부여하고, 사용자가 즉시 취해야 할 행동을 제안함.

## 2. 왜 필요한가 (Why)
- 정보 과부하 시대에 중요한 메시지를 놓치지 않게 하고, 알림 정리에 소요되는 디지털 피로도를 줄여 생산성을 높이기 위함.

## 3. 사용자 시나리오
- 사용자가 아침에 앱을 열고 현재 본인의 상황(Context: 직급, 팀, 담당 교수 등)을 설정함.
- "분류 실행" 버튼을 클릭하면 AI가 리스트업된 알림들을 분석함.
- 긴급한 알림은 빨간색으로 강조되며, 왜 긴급한지와 무엇을 해야 하는지(예: 캘린더 등록, 즉시 회신 등)를 알려줌.

## 4. 수용 기준 (Acceptance Criteria)
- [ ] 메시지는 '긴급', '오늘처리', '참고용' 중 하나로 반드시 분류되어야 한다.
- [ ] 각 분류 결과에는 20자 이내의 한 줄 요약이 포함되어야 한다.
- [ ] 사용자의 컨텍스트(직급, 소속 등)가 분류 로직에 반영되어야 한다.
- [ ] AI 응답은 정해진 JSON 형식을 준수해야 한다.
- [ ] UI는 고밀도(High Density) 대시보드 형태를 유지해야 한다.
